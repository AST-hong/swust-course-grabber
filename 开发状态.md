## 二、✅ 已完成功能

### 2.1 微信扫码登录

- **模块**: `main.py` → `LoginManager` 类
- **流程**: CAS 建 session → 获取微信 OAuth UUID → 终端二维码 → 轮询扫码状态 → callback 获取 TGC → 进入教务系统
- **持久化**: TGC cookie 保存到 `cookies.pkl`，下次自动恢复
- **命令**: `python main.py --relogin`

### 2.2 课表解析与导出

- **模块**: `course_parser.py` → `ScheduleParser`
- **解析**: 精确处理 `UICourseTable`，正确支持 rowspan/colspan
- **导出**: `export_markdown(filepath)` 输出三部分（总览表/详情/空闲时段）
- **自动同步**: 每次运行 `main.py` 都会先更新 `课表.md`，确保与当前登录账号一致
- **命令**: `python main.py -s` 或 `-s -o xxx.md`

### 2.3 多分类课程检索

- **支持分类**: 计划课程（programTask）+ 体育项目（sportTask）
- **架构**: `CATEGORIES` 类变量定义 `(event_name, section_api, choose_api, label)` 四元组
- **扩展性**: 新增分类只需在 `CATEGORIES` 列表中添加一行
- **分类筛选**: `targets.json` 中 `categories` 字段可限制搜索范围

### 2.4 选课 API（逆向 + 多端点适配）

| 分类 | 教学班 API | 选课 API |
|:---|:---|:---|
| 计划课程 | `apiPlanTaskTable` | `apiChoosePlanTask` |
| 体育项目 | `apiSportTaskTable` | `apiChooseSportTask` |

- 参数自动适配：`CT`, `TID`, `CID`, `CIDX`, `TSK`, `TT`, `ST`, `CP`
- hash 从教学班页面的 `chooseCourse()` JS 调用中提取
- 空位检测 + 自动重试

### 2.5 多条件课程筛选（三阶段流水线）

1. **课程名匹配**: 关键词模糊匹配 → 获得候选课程列表
2. **教学班过滤**: 逐一获取教学班，按 教师/时间/周次/地点 筛选
3. **空闲冲突检测**: 与教务系统实时课表比对，排除时间冲突

筛选字段（`targets.json`）:

| 字段 | 必填 | 说明 |
|:---|:---|:---|
| `keyword` | ✅ | 课程名关键词 |
| `teacher` | ❌ | 教师名（空=不限） |
| `time` | ❌ | 如 "周二第三讲"（兼容 "星期X" 和 "周X"） |
| `weeks` | ❌ | 如 "02-05" |
| `place` | ❌ | 如 "西区羽毛球场" |
| `categories` | ❌ | 限制分类，如 `["体育项目"]`，空=全部 |
| `priority` | ❌ | 数字越小越优先，默认 1 |

### 2.6 空时间/周次偏好排序

当用户不指定 `time` 或 `weeks` 时，自动按偏好排序：

- **周次优先**（高于时间）: 起始周越早越靠前
- **时间偏好**: 第2/4讲(最优) > 第3/5讲 > 第1讲(最次)
- **同条件**: 余量多的优先
- 注：指定了具体时间/周次的目标不受偏好排序影响（精确匹配即可）

### 2.7 命令行接口

| 参数 | 行为 |
|:---|:---|
| `python main.py` | 完整抢课流程（等时间→自动更新课表→登录→抢课） |
| `--relogin` / `-r` | 重新扫码登录 + 自动更新课表.md，然后退出 |
| `--schedule` / `-s` | 仅导出课表 MD |
| `--output` / `-o <file>` | 指定导出文件路径 |
| `--setup` | 交互式配置目标课程 |
| `-r -s` | 重新登录 + 导出课表 |

### 2.8 交互式目标配置

- `python main.py --setup`
- 支持追加/覆盖/清空
- 自动列出可用课程分类供选择
- 所有非必填字段回车跳过即表示"不限"

### 2.9 定时调度

- **模块**: `Scheduler` 类
- 精确等待到 `CONFIG["start_time"]`，提前 30s 验证登录
- 倒计时显示，每 10s 更新
