## 一、系统架构概览

```
用户 ──▶ matrix.dean.swust.edu.cn (教务系统)
              │
              ▼ (未登录时重定向)
         cas.swust.edu.cn/authserver (CAS 统一认证)
              │
              ├── /login          — 登录页 (微信扫码 / 短信验证码)
              ├── /weChatLogin    — 微信 OAuth 入口
              ├── /getDynamicCode — 发送短信验证码
              ├── /callback       — 微信 OAuth 回调
              └── /getKey         — RSA 公钥 (用于密码登录)
```

- **教务系统**: `matrix.dean.swust.edu.cn/acadmicManager/`
- **CAS 认证**: `cas.swust.edu.cn/authserver/`
- **认证方式**: 微信 OAuth 扫码（推荐）或 短信验证码

### 2.2 关键接口

| 步骤 | 方法 | URL | 说明 |
|------|------|-----|------|
| ① | GET | `/authserver/login?service=XXX` | 建立 session，获取 SESSION cookie |
| ② | GET | `/authserver/weChatLogin?requestUrl=XXX` | CAS 生成微信 OAuth URL |
| ③ | GET | `https://open.weixin.qq.com/connect/qrconnect?...` | 获取二维码页面，提取 UUID |
| ⑥ | GET | `https://long.open.weixin.qq.com/connect/l/qrconnect?uuid=XXX` | 轮询扫码状态 (JSONP) |
| ⑦ | GET | `/authserver/callback?code=XXXXX&state=` | 完成 CAS 认证，获取 TGC |
| ⑧ | GET | 教务系统 URL + TGC cookie | CAS 签发 service ticket |

### 2.3 WeChat 轮询状态码

| errcode | 含义 |
|---------|------|
| 408 | 等待扫码 |
| 404 | 已扫描，等待确认 |
| 405 | 确认成功，wx_code 可用 |
| 403 | 用户取消 |
| 402 | 二维码过期 |

### 2.4 Cookie 说明

| Cookie | 来源 | 说明 |
|--------|------|------|
| `route` | CAS 首次请求 | 负载均衡路由 |
| `SESSION` | CAS 首次请求 | CAS 会话标识 |
| `TGC` | CAS callback 响应 | **Ticket Granting Cookie** — 登录凭证，拿到即认证成功 |
| `ST-XXXXX` | 教务系统重定向 | Service Ticket，一次性使用 |

### 2.5 Python 实现要点

```python
import requests, re, pickle

session = requests.Session()
session.headers.update({"User-Agent": "Chrome/120.0 ..."})

# 1. 先访问登录页建立 session
session.get("http://cas.swust.edu.cn/authserver/login",
            params={"service": "教务系统URL"})

# 2. 获取微信登录入口
resp = session.get("http://cas.swust.edu.cn/authserver/weChatLogin",
                   params={"requestUrl": "登录页URL"},
                   allow_redirects=False)

# 3. 访问微信 OAuth 页获取 UUID
resp2 = session.get(resp.headers["Location"])
uuid = re.search(r'uuid=([a-zA-Z0-9]+)', resp2.text).group(1)

# 4. 轮询扫码状态
poll_url = f"https://long.open.weixin.qq.com/connect/l/qrconnect?uuid={uuid}"
resp3 = session.get(poll_url, headers={"Referer": "https://open.weixin.qq.com/"})
errcode = int(re.search(r"wx_errcode=(\d+)", resp3.text).group(1))

# 5. errcode=405 时获取 wx_code 并回调 CAS
if errcode == 405:
    wx_code = re.search(r"wx_code='([^']+)'", resp3.text).group(1)
    session.get("http://cas.swust.edu.cn/authserver/callback",
                params={"code": wx_code, "state": ""})
    # 此时 session.cookies 中已有 TGC

# 6. 携带 TGC 访问教务系统
session.get("教务系统URL")  # CAS 自动签发 ST ticket

# 7. 持久化 cookie
with open("cookies.pkl", "wb") as f:
    pickle.dump(session.cookies, f)
```

### 2.6 常见问题

**Q: callback 后重定向到 `/authserver/login` 而不是教务系统？**
A: 这是正常行为。CAS 在 callback 中设置 TGC cookie 后重定向到登录页，此时 session 已认证。直接访问教务系统 URL，CAS 检测到 TGC 会自动签发 service ticket。

**Q: 跟随 callback 重定向时 `TooManyRedirects`？**
A: 不要对 `/authserver/login` 使用 `allow_redirects=True`。获取 TGC 后直接访问教务系统 URL。

**Q: 必须先访问 `/login` 再访问 `/weChatLogin`？**
A: 是。`/weChatLogin` 需要已建立的 CAS session（SESSION cookie），否则回调时无法关联。

## 三、短信验证码登录流程

> ⚠️ 短信登录存在 Spring WebFlow 状态机问题，暂未稳定。推荐使用微信扫码。

### 3.1 流程

```
GET  /login?service=XXX          → execution: e1s1
POST /getDynamicCode {mobile}    → 发送短信 (flow 推进到 e2s1)
GET  /login?service=XXX          → 重新获取页面 (execution: e2s1)
POST /login?service=XXX          → 提交验证码登录
     {username, dynamicCode, execution, _eventId, lm, geolocation}
```

### 3.2 关键发现

Spring WebFlow 的 `execution` 参数会随状态推进而变化：
- `e1s1`: 初始页 (e1 = event 1, s1 = state 1)
- `e2s1`: 发送验证码后 (e2 = event 2)
- `e1s2`/`e2s2`: 登录失败后的新状态

**发送 getDynamicCode 后必须重新获取页面拿到最新的 execution 值**，否则登录提交会被拒绝。

## 四、课表解析

### 4.1 HTML 结构

课表页面 URL: `event=chooseCourse:courseTable`

```html
<table class="UICourseTable">
  <thead>
    <tr>
      <td colspan="2"></td>
      <td>星期一</td> <td>星期二</td> ... <td>星期天</td>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td rowspan="2">上午</td>      <!-- 时段标签 (rowspan 跨2行) -->
      <td>第一讲</td>                <!-- 节次标签 -->
      <td></td>                     <!-- 周一内容 -->
      <td>                          <!-- 周二内容 -->
        <div class="lecture">
          <span class="course">计算机网络</span> -
          <span class="teacher">李春燕</span>
          <br><span class="week">01-10(2)</span>
          <br><span class="place">东1212</span>
        </div>
      </td>
      <td></td> ... <td></td>        <!-- 周三~周日 -->
    </tr>
    <tr>
      <!-- 第二行：col 0 被上一行 rowspan 占据，少一列 -->
      <td>第二讲</td>
      <td></td> ... <td></td>
    </tr>
    ...
  </tbody>
</table>
```

### 4.2 解析要点

1. **表头列映射**：`<td colspan="2">` 占2列 → 周一从第2列开始
   ```
   header索引:  [0(占2列)] [1=周一] [2=周二] [3=周三] ...
   对应body列:  [0] [1]    [2]      [3]      [4]
   ```
   body 列号 = header 索引 + 1

2. **rowspan 处理**：`<td rowspan="2">` 跨2行 → 第二行少一列
   ```python
   if cells[0].get("rowspan"):
       rowspan_skip = False   # 完整行
   else:
       rowspan_skip = True    # 缺第一列
   ```

3. **课程提取**：通过 class 精确匹配
   ```python
   div = cell.find("div", class_="lecture")
   name = div.find("span", class_="course").get_text()
   teacher = div.find("span", class_="teacher").get_text()
   weeks = div.find("span", class_="week").get_text()
   place = div.find("span", class_="place").get_text()
   ```

### 4.3 节次映射

| 讲次 | 小节号 | 时段 |
|------|--------|------|
| 第一讲 | 1-2 | 上午 |
| 第二讲 | 3-4 | 上午 |
| 第三讲 | 5-6 | 下午 |
| 第四讲 | 7-8 | 下午 |
| 第五讲 | 9-10 | 晚上 |
| 第六讲 | 11-12 | 晚上 |

## 五、教务系统 URL 模式

教务系统使用 `event` 参数进行页面路由：

### 5.1 页面入口

| 页面 | URL |
|------|-----|
| 学生门户 | `?event=studentPortal:DEFAULT_EVENT` |
| 网上选课 | `?event=chooseCourse:DEFAULT_EVENT` |
| 个人课表 | `?event=chooseCourse:courseTable` |
| 计划课程 | `?event=chooseCourse:programTask&CT=1` |
| 体育项目 | `?event=chooseCourse:sportTask&CT=1` |
| 学期学费 | `?event=chooseCourse:chargeByTerm` |
| 学生档案 | `?event=studentProfile:DEFAULT_EVENT` |
| 课程成绩 | `?event=studentProfile:courseMark` |
| 考试安排 | `?event=studentPortal:examTable` |
| 上课安排 | `?event=studentPortal:courseTable` |
| 教学评价 | `?event=evaluateOnline:DEFAULT_EVENT` |

### 5.2 AJAX API 端点（选课）

选课系统按课程分类使用不同的 API 端点：

| 分类 | event 参数 | 教学班 API | 选课 API |
|:---|:---|:---|:---|
| 计划课程 | `programTask` | `apiPlanTaskTable` | `apiChoosePlanTask` |
| 体育项目 | `sportTask` | `apiSportTaskTable` | `apiChooseSportTask` |

**教学班查询** (`apiPlanTaskTable` / `apiSportTaskTable`):

```
POST ?event=chooseCourse:apiPlanTaskTable
Data: TID=<term_id>, CID=<course_id>, seed=<timestamp>
Referer: ?event=chooseCourse:programTask&CT=1
```

返回 HTML 片段，包含 `<tr class="editRows">` 行，每行一个教学班。
选课链接通过 `chooseCourse()` JS 调用嵌入：

```javascript
chooseCourse('<course_id>','<course_idx>','<term_id>','<task_type>','<task_id>','<hash>')
```

**提交选课** (`apiChoosePlanTask` / `apiChooseSportTask`):

```
POST ?event=chooseCourse:apiChoosePlanTask
Data:
  CT=<1>
  TID=<term_id>
  CID=<course_id>
  CIDX=<course_idx>
  TSK=<task_id>        # 计划课程为纯数字，体育项目为 "tid,cid,idx" 复合格式
  TT=<task_type>        # 计划课程=P, 体育项目=T
  ST=<hash>             # 从 chooseCourse() 中提取，有时效性
  CP=<prop>             # 课程属性（计划课程有值，体育项目为空）
  seed=<timestamp>
Referer: ?event=chooseCourse:programTask&CT=1  (或 sportTask)
```

响应 JSON：`{"success":true}` 或 `{"success":false,"reason":"..."}`

## 六、课程列表解析

### 6.1 HTML 结构

课程列表页（`programTask` / `sportTask`）以 `div.courseShow` 渲染每门课程：

```html
<div cid="116662" class="courseShow clearfix" prop="2">
  <span class="courseName">变形监测与数据处理</span>
  <span class="courseType">限选</span>
  <span class="credit">1.5</span>
</div>
```

### 6.2 提取字段

| 属性/元素 | 字段 | 说明 |
|:---|:---|:---|
| `div[cid]` | `course_id` | 课程唯一 ID |
| `div[prop]` | `prop` | 课程属性（1=必修, 2=限选），体育项目无此属性 |
| `span.courseName` | `name` | 课程名称 |
| `span.courseType` | `type` | 课程类型 |
| `span.credit` | `credit` | 学分 |
| 页面 URL 参数 | `event_name` | 所属分类（programTask / sportTask） |
