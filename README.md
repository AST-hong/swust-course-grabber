## 功能

- 🔑 **微信扫码登录** — 终端显示二维码，扫码即登，cookie 持久化免重复登录
- 📅 **课表自动同步** — 每次运行自动拉取最新课表并导出为 Markdown 文档
- 🔍 **多分类课程检索** — 支持「计划课程」「全校通选课」「体育项目」「重新学习」「补选低年级课程」五个分类
- 🎯 **多条件筛选** — 按教师、上课时间、周次、上课地点精准筛选教学班
- 📊 **偏好排序** — 空时间/周次时自动按周次早晚 → 讲次偏好 → 余量排序
- ⏰ **定时启动** — 设定时间点自动开始，倒计时显示
- 🔄 **失败重试** — 选课失败自动重试，可配置间隔和次数
- 🍪 **免重复登录** — TGC cookie 自动保存，下次运行无需重新扫码

## 环境要求

- **Python** >= 3.8
- 终端需支持二维码显示（Windows Terminal / PowerShell / CMD 均可）
- 网络可访问西南科技大学 CAS 认证服务器和教务系统（既已挂载aTrust）

## 快速开始

### 1. 克隆仓库

```powershell
git clone https://github.com/AST-hong/swust-course-grabber.git
cd swust-course-grabber
```

### 2. 安装依赖

```powershell
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 3. 配置选课信息

**推荐方式：交互式配置**

```powershell
python main.py --setup
```

按提示输入即可，配置保存到 `targets.json`。支持追加、覆盖、清空，以及按课程分类筛选。

**备选方式：直接编辑配置文件**

编辑 `targets.json`：

```json
[
  {
    "keyword": "乒羽俱乐部",
    "teacher": "刘瑛",
    "priority": 1,
    "time": "周一第一讲",
    "weeks": "02-05",
    "place": "西区羽毛球场",
    "categories": ["体育项目"]
  },
  {
    "keyword": "大数据",
    "teacher": "",
    "priority": 2,
    "time": "",
    "weeks": "",
    "place": "",
    "categories": []
  }
]
```

| 字段 | 必填 | 说明 |
|:---|:---|:---|
| `keyword` | ✅ | 课程名关键词（模糊匹配） |
| `teacher` | ❌ | 教师名，空=不限 |
| `time` | ❌ | 如 "周二第三讲"，空=不限（按偏好自动排序） |
| `weeks` | ❌ | 如 "02-05"，空=不限（按起始周早晚自动排序） |
| `place` | ❌ | 如 "西区羽毛球场"，空=不限 |
| `categories` | ❌ | 限制分类，如 `["体育项目"]`，空=全部 |
| `priority` | ❌ | 数字越小越优先，默认 1 |

> **时间/周次为空时**：脚本会按「周次越早越优先 → 第2/4讲最优 → 余量多优先」自动选择。

### 4. 设置抢课时间

编辑 `main.py` 顶部的 `CONFIG` 字典，修改以下参数：

```python
CONFIG = {
    # 抢课开始时间 (年, 月, 日, 时, 分, 秒)
    "start_time": datetime(2026, 6, 25, 9, 0, 0),

    # 提前多久开始准备 (秒) — 登录 + 拉取课表
    "prepare_advance": 300,

    # 重试配置
    "max_retries": 50,        # 每门课最大重试次数
    "retry_interval": 0.1,    # 重试间隔 (秒)
}
```

| 参数 | 说明 |
|:---|:---|
| `start_time` | 选课系统开放时间，脚本会倒计时等待 |
| `prepare_advance` | 提前启动秒数，确保登录和课表拉取在抢课前完成 |
| `max_retries` | 每门课程的最大尝试次数 |
| `retry_interval` | 每次失败后等待秒数再重试 |
| `ct` | 选课学期 CT 参数（当前为 2，如页面 URL 中 CT 值变化需同步修改） |

### 5. 运行

```powershell
python main.py --setup       # 交互式配置目标课程
python main.py               # 正常启动（自动更新课表，进入定时抢课）
python main.py --relogin     # 重新扫码登录 + 更新课表.md，然后退出
python main.py -r            # 同上（简写）
python main.py --schedule    # 仅导出课表为 Markdown 文件
python main.py -r -s         # 重新登录 + 导出课表（组合使用）
python main.py -s -o 我的课表.md  # 导出课表并指定文件名
```

### 6. 课表文件

每次运行 `main.py` 都会自动将当前课表更新到 `课表.md`，包含三部分：

- **课表总览** — 以表格形式展示一周课程
- **已选课程详情** — 每门课的时间、教师、周次、教室（一周多次课各自列出）
- **空闲时段** — 可用于排课的空闲讲次

### 7. 选课优先级逻辑

每次运行时，脚本按以下层级依次决策：

#### 第一层：目标课程优先级

`targets.json` 中每门课的 `priority` 字段决定处理顺序（数字越小越先处理）。
优先级高的课程抢到后才处理下一门。

#### 第二层：单门课程的教学班排序

对每门目标课程，先通过三阶段筛选（课程名匹配 → 教学班过滤 → 冲突检测），
再对剩余可选教学班排序。排序规则取决于你在配置中是否指定了时间/周次：

**指定了时间和周次** — 精确匹配，只按余量排序（席位多的优先）。

**未指定时间/周次** — 按以下偏好自动排序：

```
优先级: 周次早晚  >  讲次偏好  >  余量多少
```

| 排序维度 | 规则 | 说明 |
|:---|:---|:---|
| ① 周次 | 起始周越早越靠前 | 如 02-05 周优先于 06-09 周 |
| ② 讲次 | 第2/4讲 > 第3/5讲 > 第1讲 | 仅当周次相同时比较 |
| ③ 余量 | 剩余席位多优先 | 前两项都相同时比较 |

> **说明**：只指定了时间但未指定周次，则跳过讲次偏好比较（时间已精确筛选），
> 只按周次早晚 + 余量排序。反之同理。

#### 第三层：冲突检测

排序前所有教学班都会先与教务系统实时课表比对，时间冲突的班直接排除。

#### 第四层：重试

按排序结果依次尝试提交选课，成功则停止。全部失败则等待后重新获取
教学班数据（hash 会刷新）再次尝试，直至达到最大重试次数。

├── cookies.pkl           # 登录凭证缓存（自动生成，Git 忽略）
├── 课表.md               # 课表文件（每次运行自动更新，Git 忽略）
├── README.md             # 使用说明
├── 技术说明.md           # 技术文档（认证流程、API 逆向、课表解析）

## Cookie 管理

- 首次登录后自动保存 `cookies.pkl`
- 下次运行自动恢复，无需重新扫码
- 换账号：`python main.py --relogin`（自动更新课表）
- 手动清除：删除 `cookies.pkl`

## 注意事项

- 首次使用需在终端扫码一次
